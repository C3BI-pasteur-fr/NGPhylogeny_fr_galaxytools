----------------

Input: multi-FASTA file
FASTA is text-file format for representing nucleotide sequences or amino acids sequences. The sequence name starts with ">" and the following lines contain sequences. A multi-FASTA file contains multiple FASTA format sequences.

----------------

MAFFT:


- Input: Multifasta file from Step 1
- Output: Mutli-Fasta file with aligned sequences

parameters:

- Mafft flavour: auto
Automatically selects an appropriate strategy from L-INS-i, FFT-NS-i and FFT-NS-2, according to data size.
- Gap Extension: (--ep 0.0) allows longer indels and thus is more robust to unusual evolutionary events, such as domain-level indels.

-----------------

Analyse fasta file to detect witch type of sequences is contained in the file (dna or protein)

-----------------

Noisy:


- Input: Aligned sequences in Fasta file (from Step 2)
- Output: Multifasta aligned sequences file

parameters:

- cut-off: 0.4
Ensure to removing most noisy sites and without loosing to much phylogenetic signal
- Ordering method: Neigthbor-Net
Fast method that grouping taxa close to each other that have a short distance
- Distance method: Hamming
Use distance matrix calculated by hamming method. (Distance between two strings of equal length is the number of positions at which the corresponding symbols are different)

-----------------

Convertion Fasta to Phylip
 Input: Aligned sequences in Fasta file (from Step 4)
Output: Sequential sequences Phylip File

-----------------

FastME:


Input: Phylip file (from Step 5)
Output: Tree Newick file

parameters:

- Evolutionary model Use HKY85 or GTR according to sequences type
  • DNA: HKY85
  • Protein: GTR
- Equilibrium frequencies: YesFrequencies are globally counted from the nucleotides alignment or defined by the proteic substitution model. By checking the box, frequencies are pairwise estimated by counting the nucleotides or estimated by counting the amino-acids in the alignment.
- Gamma distributed rates across sites: Yes Use gamma parameter
- Gamma distribution: 1
- Remove gap strategy: Pairwise deletion of gaps
- Distance algorithm: BoiNJ Use BioNJ algorithm to calculate distances
- Tree refinement: BalME SPR Search optimized tree topology with Subtree-Pruning-Regrafting (SPR)

----------------

PhyML:


Input: Phylip file (from Step 5)
Output: Tree Newick file

parameters:

- Substitution model:
Use HKY85 or GTR according to sequences type
  • DNA: HKY85
  • Protein: GTR
- Proportion of invariable sites: Estimated
- Equilibrium frequencies: ML/Model
- Transition/transversion ratio (DNA models): Estimated
- Parameter of the gamma model: Estimated
- Distance algorithm: BoiNJ Use BioNJ algorithm to calculate distances
- Tree topology search: SPR Search optimized tree topology with Subtree-Pruning-Regrafting (SPR)
- Optimise topology: Tree topology, Branch length, Rate parameter - Statistical tests for branch support: SH-like
